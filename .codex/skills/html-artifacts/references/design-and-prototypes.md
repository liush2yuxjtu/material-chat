# Design & Prototypes

Use HTML for living design systems, component contact sheets, interaction prototypes, and clickable flows.

Render design tokens as themselves: color swatches, real typography specimens, visible spacing, radii, shadows, and motion. Add copy controls for token names and values, and source tokens from the codebase rather than inventing them.

For component sheets, show one component across all sizes, intents, and states, including loading, empty, error, focus, hover, and disabled states. Display the props used for each variant.

For animation prototypes, isolate the target, expose meaningful parameters with live controls, provide replay, visualize easing where useful, and continuously generate copyable CSS or JavaScript.

For clickable flows, implement working navigation across 3–6 representative screens and provide a screen tray for direct jumps. Optimize for validating sequence and interaction shape, not pixel perfection.
