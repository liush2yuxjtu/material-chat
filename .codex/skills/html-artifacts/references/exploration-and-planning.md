# Exploration & Planning

Use this reference for option comparisons, visual-direction exploration, and implementation plans.

## Side-by-side comparisons

Render genuinely different options in parallel columns with identical internal structure. Include the artifact or code for each option, a two-column pros/cons table, comparable hard metrics, and a recommendation that selects one approach.

Avoid sequential sections that force the reader to remember earlier options, superficial variations of one approach, or refusing to recommend.

## Visual design exploration

Show 4–6 meaningfully different rendered directions in a responsive grid. Vary layout, density, hierarchy, tone, and interaction model before changing surface color. Each direction should include a short tradeoff caption and may expand for inspection.

## Implementation plans

Use a visual milestone strip, an inline-SVG data-flow diagram, relevant UI mockups, annotated load-bearing code, a risk/likelihood/mitigation table, and an explicit out-of-scope section. Show the shape of the change rather than listing every touched file.

The data-flow diagram, risk table, and exclusions are mandatory when applicable.
