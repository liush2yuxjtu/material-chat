# Decks

Use a self-contained HTML deck when the user explicitly requests slides or a presentation intended to be narrated.

Use one `<section>` per slide, show only the active slide, support left/right arrows and space, include a slide counter, and support fullscreen. Preserve a 16:9 presentation surface with letterboxing rather than allowing arbitrary layout shifts.

Keep one idea per slide, use large type, and minimize words. Vary slide composition instead of turning every slide into the same card. Prefer charts, diagrams, code, quotes, and strong visual hierarchy over bullet-heavy pages.

Optional features include thumbnail navigation, presenter notes, and a light/dark toggle. Avoid ornamental transitions and dense reference material that belongs in a report instead.
