# Matching the User's Style

Avoid generic AI-looking output. Prefer restraint, real typography, and color that communicates status or meaning rather than decoration.

When a product or codebase already has a visual system, inspect its CSS variables, Tailwind configuration, theme files, typography, spacing, radii, shadows, and component patterns. Build or reuse a `design-system.html` reference and apply those tokens to later artifacts.

Without an existing system, use a calm neutral background, readable ink, one restrained accent, semantic warning/error/success colors, 16–18px body text, 60–75ch line length, and approximately 1.5–1.6 line height. Documents may use a serious serif stack; tools and editors should use a compact system sans-serif.

Avoid gradients for mood, cards everywhere, emoji section headers, arbitrary violet palettes, glass effects, placeholder logos, centered-everything layouts, and unnecessary framework-shaped components. If a frontend-design skill or repository design system exists, follow it instead of these defaults.
